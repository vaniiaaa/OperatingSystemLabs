ООП
Это способ написания программ, при котором вся программа
делится на отдельные объекты, которые обладают своими свойствами
и возможностями

ОО декомпозиция
Это метод разбиения сложных систем задач на более простые

Дизайн ПО
Это план по написанию будущей программы, а именно как объединять
различные компоненты, чтобы они были удобны для дальнейших правок

Число Милера 7
Человек не может одновременно быть сконцентрирован более чем на 7 задачах +-
Поэтому в айти этот подход рассматривается как ограничение при организации проектов
Примеры:
Ограничение числа списка типов параметров в функциях
Ограничение на количество методов в классах
Ограничение на выполнение задач для людей
Ограничение на количесвто элементов в выпадающих списках

5 Признаков сложных систем по Гради Бучу
1. Каждая задача имеет относительно иерархическую структуру, то есть состоят из подсистем, которые сами имеют подсистемы
2. Выбор какие задачи считать примитивными неоднозначен
3. Подсистемы больше взаимодействуют между друг другом внутри, чем между внешними
4. Все системы состоят из ограниченного числа комбинированных подсистем
5. Любая работающая система является результатом развития работающей более простой системы
Примеры: Веб-приложения делятся на front, back, базы данных, которые сами в свою очередь тоже делятся 
на состовляющие, каждый уровень может быть спроетирован относительно независимо.
Операционные системы Драйверы устройств, подсистема управления процессами, подсистема управления памятью, сетевая подсистема.
Подсистема управления процессами: Планировщик (Scheduler), механизмы межпроцессного взаимодействия (IPC)
Сама ОС работает на аппаратном обеспечении, а поверх нее запускаются пользовательские приложения
